หุ่นยนต์ เซนเซอร์ และระบบหยุดอย่างปลอดภัย
Robots, Sensors and Fail-Safe Systems

คำถามนำ / Big question: ระบบอัตโนมัติควรทำอย่างไรเมื่อเซนเซอร์ไม่แน่ใจหรือพบสถานการณ์ที่ไม่คาดคิด?

ไปที่ / Jump to: เรียนรู้ · ตัวจำลอง · หยุดอย่างปลอดภัย · แบบฝึกหัด · สำหรับผู้ใหญ่

เป้าหมาย / Learning goals

  • อธิบาย Input–Process–Output ของระบบหุ่นยนต์อย่างง่าย
  • แยกค่าจากเซนเซอร์ออกจากความจริงทั้งหมด
  • ระบุความผิดพลาด เช่น เซนเซอร์เปียก สกปรก บัง หรืออ่านค่าไม่ได้
  • ออกแบบกติกา stop–alert–human เมื่อความเสี่ยงหรือความไม่แน่นอนสูง

คำศัพท์ / Vocabulary

sensor — เซนเซอร์
อุปกรณ์ที่วัดสิ่งหนึ่ง เช่น แสง ความชื้น หรือระยะทาง
input — ข้อมูลเข้า
ค่าหรือสัญญาณที่ระบบได้รับ
process — การประมวลผล
กฎที่ใช้แปลข้อมูลและเลือกการกระทำ
output — ผลลัพธ์
การแสดงผล การแจ้งเตือน หรือการทำงานของอุปกรณ์
fail-safe — ระบบปลอดภัยเมื่อผิดพลาด
เมื่อข้อมูลไม่น่าเชื่อถือ ระบบหยุดหรือลดความเสี่ยงและเรียกมนุษย์

กรอบ SAFE / SAFE frame

  1. S — Sense: รับข้อมูลจากเซนเซอร์
  2. A — Assess: ตรวจช่วงค่า คุณภาพข้อมูล และความเสี่ยง
  3. F — Fail safely: หยุด ปิด หรือลดการทำงานเมื่อไม่แน่ใจ
  4. E — Escalate: แจ้งมนุษย์ที่รับผิดชอบและบันทึกเหตุผล

ขอบเขต: ตัวจำลองนี้ใช้ข้อมูลสมมติในหน้าเว็บ ไม่เชื่อมต่ออุปกรณ์จริง ไม่ควบคุมปั๊มน้ำ กล้อง ไมโครโฟน หรือติดตามเด็ก

กิจกรรม 1: Input–Process–Output simulator

ระบบรดน้ำต้นไม้สมมติอ่านค่าความชื้น 0–100 แต่ค่าเดียวอาจผิดได้จากเซนเซอร์เปียก สกปรก หลุด หรือวางผิดตำแหน่ง

สถานะเซนเซอร์ / Sensor condition

กิจกรรม 2: เลือกกติกาหยุดอย่างปลอดภัย / Choose a fail-safe rule

เมื่อเซนเซอร์อ่านค่าไม่ได้หรือถูกบัง ระบบควรทำอย่างไร?

แบบฝึกหัด / Quick check

1) เซนเซอร์บอกอะไร?
2) เมื่อข้อมูลขัดแย้งหรือหายไป ระบบเสี่ยงควรทำอย่างไร?
3) ควรใช้บทเรียนนี้ควบคุมอุปกรณ์จริงหรือเฝ้าติดตามเด็กหรือไม่?

สรุป / Summary

หุ่นยนต์รับข้อมูล ประมวลผล และสร้างผลลัพธ์ แต่เซนเซอร์มีข้อจำกัด ระบบที่รับผิดชอบต้องตรวจคุณภาพข้อมูล อธิบายกฎ และหยุดอย่างปลอดภัยเมื่อไม่แน่ใจ
