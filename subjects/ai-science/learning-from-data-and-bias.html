AI-03 · บทเรียน 3 / Lesson 3

แบบจำลองเรียนรู้จากข้อมูลและอคติ
Learning from Data and Bias

คำถามนำ / Big question: ถ้าแบบจำลองได้เห็นตัวอย่างไม่ครบหรือไม่สมดุล ผลลัพธ์จะยุติธรรมและน่าเชื่อถือได้อย่างไร?

ไปที่ / Jump to: เรียนรู้ · ฝึกแบบจำลอง · เปรียบเทียบข้อมูล · แบบฝึกหัด · สำหรับผู้ใหญ่

เป้าหมาย / Learning goals

  • อธิบายว่าแบบจำลองเรียนจาก ตัวอย่างที่มีป้ายกำกับ / labelled examples
  • สังเกตว่าชุดข้อมูลที่ไม่สมดุลอาจทำให้ความแม่นยำต่างกันระหว่างกลุ่มตัวอย่าง
  • ใช้กรอบ FAIR — Find gaps, Add variety, Inspect errors, Report limits
  • เข้าใจว่าผลของแบบจำลองเป็นการคาดเดา ไม่ใช่ความจริงแน่นอนหรือคำตัดสินคุณค่าของคน

คำศัพท์ / Vocabulary

training data — ข้อมูลฝึก
ตัวอย่างที่ใช้ให้แบบจำลองเรียนรู้รูปแบบ
label — ป้ายกำกับ
คำตอบที่บอกว่าตัวอย่างนั้นอยู่ในหมวดใด
model — แบบจำลอง
กฎหรือระบบที่เรียนรู้รูปแบบจากตัวอย่างเพื่อคาดเดาตัวอย่างใหม่
bias — อคติของระบบ
ความเอนเอียงที่ทำให้ผลลัพธ์ผิดพลาดหรือไม่เป็นธรรมต่อบางกลุ่ม โดยอาจเกิดจากข้อมูล การออกแบบ หรือวิธีใช้

หลักสำคัญ / Key idea

แบบจำลองไม่ “รู้” โลกเหมือนคน แต่ค้นหารูปแบบจากตัวอย่างที่เราให้ ถ้าตัวอย่างมีน้อย ซ้ำกัน มีป้ายผิด หรือขาดความหลากหลาย แบบจำลองอาจเรียนผิดและทำพลาดซ้ำ

  1. FIND GAPS — หาช่องว่าง: ตัวอย่างแบบใดหายไป?
  2. ADD VARIETY — เพิ่มความหลากหลาย: เพิ่มสภาพ สี รูปร่าง ภาษา หรือบริบทที่เกี่ยวข้องกับงาน
  3. INSPECT ERRORS — ตรวจข้อผิดพลาด: แบบจำลองพลาดกับตัวอย่างแบบใดมากกว่า?
  4. REPORT LIMITS — บอกข้อจำกัด: อธิบายสิ่งที่แบบจำลองยังทำไม่ได้และให้คนตรวจงานสำคัญ

ขอบเขตบทเรียน: ใช้เพียงบัตรผลไม้และรูปร่างสมมติ ไม่ขอชื่อ ภาพใบหน้า เสียง เชื้อชาติ ศาสนา ความพิการ สุขภาพ ตำแหน่ง หรือข้อมูลจริงของผู้เรียน และไม่ใช้ AI ตัดสินคุณค่าหรือโอกาสของบุคคล

กิจกรรม 1: ฝึกแบบจำลองผลไม้สมมติ / Fictional fruit trainer

เลือกชุดข้อมูลฝึก แล้วกดทดสอบ แบบจำลองง่ายนี้ใช้กฎจากตัวอย่างสีและรูปทรง ไม่มีภาพหรือข้อมูลจริงถูกอัปโหลด

เลือกชุดข้อมูล / Choose a dataset

กิจกรรม 2: นักสืบความเป็นธรรม / Fairness detective

อ่านรายงานผลสมมติ แล้วเลือกการปรับปรุงที่เหมาะสมที่สุด

แบบจำลองอ่านป้ายสองภาษาได้ถูกต้อง: ภาษาไทย 9/10 แต่ภาษาอังกฤษ 4/10 เพราะข้อมูลฝึกมีภาษาอังกฤษน้อยมาก ควรทำอย่างไร?

แบบฝึกหัด / Quick check

1) ข้อมูลฝึกมีหน้าที่อะไร?
2) ข้อใดเป็นสัญญาณว่าควรตรวจอคติ?
3) งานสำคัญที่กระทบคนควรทำอย่างไร?

สรุปและหลักฐาน / Exit ticket and evidence

เขียนหนึ่งประโยค: “ชุดข้อมูลที่ดีขึ้นควร… เพราะ…” / Write one sentence: “A better dataset should… because…”

สำหรับผู้ใหญ่ / Adult note

เน้นว่า “อคติ” ในบทนี้หมายถึงรูปแบบความผิดพลาดของระบบ ไม่ใช่การติดป้ายเด็ก หลีกเลี่ยงข้อมูลส่วนบุคคลและลักษณะอ่อนไหว ใช้ตัวอย่างสมมติ ให้ผู้เรียนวิจารณ์ระบบไม่ใช่วิจารณ์คน และย้ำว่าการเพิ่มข้อมูลอย่างเดียวไม่พอ ต้องตรวจป้ายกำกับ วัตถุประสงค์ ผลกระทบ และสิทธิอุทธรณ์ด้วย
