AI-04 คู่มือผู้สอน
Patterns, Features and Edge Cases

เวลาและผลลัพธ์

60–90 นาที ผู้เรียนอธิบาย pattern/feature แยก training จาก test examples สังเกต edge case และปรับกฎอย่างมีหลักฐาน

แผนการสอน

  1. 10 นาที: สำรวจคำศัพท์ด้วยบัตรสิ่งของสมมติ
  2. 20 นาที: สร้างกฎจาก training examples โดยให้เหตุผลเรื่อง feature
  3. 15 นาที: เปิด test examples ที่เก็บไว้และบันทึกผลทุกข้อ
  4. 15 นาที: วิเคราะห์ edge case และเขียนข้อจำกัด
  5. 10–20 นาที: ใบงาน แบบประเมิน และ exit ticket

แนวคำตอบ

กฎที่ดีใช้คุณลักษณะซึ่งสัมพันธ์กับงาน เช่น “เก็บของภายในและปิดได้” ไม่ใช่สีหรือขนาดเพียงอย่างเดียว เมื่อถุงผ้าทำให้กฎ “แข็งและมีฝา” ผิด ผู้เรียนควรบันทึกผล ทบทวน feature และทดสอบกฎใหม่ด้วยตัวอย่างใหม่ ไม่เปลี่ยนคำตอบจริงให้เข้ากฎ

Accessibility และ differentiation

ให้ตอบด้วยการพูด บัตรคำ ภาพ สัญลักษณ์ การเขียนตามคำบอก หรืออุปกรณ์สื่อสารได้ กิจกรรมเว็บใช้ radio buttons และ buttons แบบ native ไม่มี drag-only หรือ timer ให้อ่านตัวอย่างทีละใบและพักได้

ความปลอดภัยและจริยธรรม

  • ใช้สิ่งของสมมติหรือของสะอาดน้ำหนักเบาที่ผู้ใหญ่ตรวจแล้ว
  • ไม่ขอชื่อ ภาพ เสียง ตำแหน่ง สุขภาพ ผลการเรียน หรือข้อมูลครอบครัว
  • ห้ามใช้กฎจำแนก ประเมิน หรือจัดอันดับบุคคล
  • อธิบายว่า classifier เป็นแบบจำลองเพื่อเรียนรู้ ไม่ใช่ระบบรับรองความปลอดภัย คุณภาพ หรือความเหมาะสมของผลิตภัณฑ์

Rubric 4 ระดับ

ระดับหลักฐาน
4 สร้างสรรค์เลือก feature ที่เกี่ยวข้อง แยก test set อธิบาย edge case ปรับกฎและข้อจำกัดครบ
3 ประยุกต์สร้างและทดสอบกฎได้ พร้อมอธิบายข้อผิดพลาดส่วนใหญ่
2 กำลังพัฒนาระบุ pattern หรือ feature ได้ แต่ยังปะปน training/test หรือซ่อนข้อผิดพลาด
1 เริ่มต้นต้องการแบบอย่างและคำถามชี้นำทีละขั้น
