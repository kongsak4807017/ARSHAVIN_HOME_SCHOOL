คู่มือผู้สอน AI-06 / Teacher Guide

เวลา / Duration: 60–90 นาที · รูปแบบ: สนทนา ทดลองตัวแนะนำสมมติ ใบงาน และสะท้อนผล

จุดประสงค์

  • ผู้เรียนอธิบายได้ว่าอันดับเป็นผลจากเป้าหมาย เกณฑ์ ข้อมูล และน้ำหนัก
  • ผู้เรียนเปลี่ยนเกณฑ์และสังเกตว่าอันดับเปลี่ยน
  • ผู้เรียนระบุองค์ประกอบของ transparency, user control และ fairness review
  • ผู้เรียนยืนยันว่าไม่ควรใช้ recommender จัดอันดับหรือ profiling เด็ก

ลำดับการสอน

  1. 10 นาที: เปรียบเทียบ “คำแนะนำ” กับ “คำสั่ง” โดยใช้ตัวอย่างกิจกรรมสมมติ
  2. 15–20 นาที: อ่านตารางข้อมูลและทดลองเปลี่ยน priority ในตัวแนะนำ
  3. 15 นาที: ใช้ CLEAR วิเคราะห์ว่าเกณฑ์ใดมองเห็นได้ เกณฑ์ใดอาจกีดกัน
  4. 20–30 นาที: ทำใบงานสองหน้าและอภิปรายข้อจำกัด
  5. 5–10 นาที: แบบประเมินและ exit statement: “อันดับนี้ขึ้นกับ…”

แนวคำตอบสำคัญ

  • ไม่มีอันดับที่ดีที่สุดสำหรับทุกคน เพราะเป้าหมายและข้อจำกัดต่างกัน
  • เมื่อให้น้ำหนัก teamwork สูง เกมบัตรคำกลุ่มอาจขึ้นอันดับแรก; เมื่อให้น้ำหนัก access สูง รายการอื่นอาจเสมอหรือขึ้นอันดับ
  • ผู้ใช้ควรดูเกณฑ์ เปลี่ยนหรือปิดคำแนะนำ ดูทางเลือกอื่น และขอคำอธิบายได้
  • ห้ามใช้ชื่อ คะแนน สุขภาพ พฤติกรรม ฐานะ ภาพ เสียง หรือข้อมูลครอบครัวจัดอันดับเด็ก

ความเข้าใจผิดที่ควรแก้

  • “อันดับ 1 ดีที่สุดเสมอ” — ไม่จริง อันดับสะท้อนกฎและน้ำหนักที่เลือก
  • “ระบบเป็นกลางเพราะใช้ตัวเลข” — ตัวเลขยังมาจากการเลือกเกณฑ์ ข้อมูล และน้ำหนักของมนุษย์
  • “personalization ต้องเก็บข้อมูลมาก” — ไม่จำเป็น กิจกรรมนี้ใช้การเลือกชั่วคราวในเครื่องโดยไม่สร้างโปรไฟล์

Accessibility และการมีส่วนร่วม

ใช้ native select, radio และ button; ไม่มี timer หรือ drag-only interaction. ผู้เรียนตอบด้วยคำพูด บัตรคำ ภาพ ลูกศร การเขียนตามคำบอก หรือ AAC ได้ ผู้สอนอ่านตารางออกเสียงและให้สำเนาขยายได้ หลีกเลี่ยงการให้ผู้เรียนเปิดเผยความชอบส่วนตัวหากไม่ต้องการ

Privacy, fairness และ safeguarding

  • ใช้เฉพาะกิจกรรมและข้อมูลสมมติ ไม่ป้อนข้อมูลผู้เรียนจริง
  • ห้ามใช้ผลลัพธ์จัดกลุ่มความสามารถ ให้คะแนน วินิจฉัยสุขภาพ ตัดสินความประพฤติ หรือกำหนดโอกาส
  • ไม่ใช้ระบบสร้างแรงกดดันทางสังคมหรือทำให้เด็กแข่งขันเพื่ออันดับ
  • หากผู้เรียนเปิดเผยการถูกติดตาม เลือกปฏิบัติ หรือไม่ปลอดภัย ให้หยุดกิจกรรม รักษาความเป็นส่วนตัว และส่งต่อผู้ใหญ่ตามระบบคุ้มครองเด็ก

Rubric 4 ระดับ

ระดับความเข้าใจหลักฐานความรับผิดชอบ
4เชื่อม goal–criteria–weight–ranking และ trade-off ได้ชัดทดลองเปลี่ยนน้ำหนักและอธิบายผลออกแบบ transparency, user control, fairness review และ no-child-profiling ครบ
3อธิบายว่าอันดับขึ้นกับเกณฑ์ใช้ข้อมูลตารางสนับสนุนข้อสรุปให้ผู้ใช้เลือกและไม่ใช้ข้อมูลเด็ก
2ระบุบางเกณฑ์ได้แต่ยังคิดว่าอันดับเป็นคำตอบเดียวหลักฐานยังไม่เชื่อมกับข้อสรุปต้องเตือนเรื่อง privacy หรือ user control
1สับสนระหว่าง recommendation กับคำสั่งไม่มีหลักฐานหรือเดาจากความชอบเสนอ profiling/จัดอันดับเด็กหรือซ่อนเกณฑ์
